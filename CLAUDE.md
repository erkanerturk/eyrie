This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Eyrie is a modular macOS 26+ menu bar app (SwiftUI, Liquid Glass). Six feature modules — AwakeKit (keep-awake), FocusKit (pomodoro), AudioShareKit (multi-device audio), DisplayKit (DDC/CI brightness), StatsKit (CPU/memory/network stats), NetKit (network identity) — live as local Swift Packages under `Packages/`, plus `EyrieCore` (shared protocol, services, glass UI components). The app target in `App/` is only a shell.

## Commands

The Xcode project is **generated by XcodeGen** — never edit `Eyrie.xcodeproj` (it's gitignored). Edit `project.yml`, then:

```bash
xcodegen generate
```

On this machine `xcode-select` points at the CommandLineTools, so xcodebuild needs a `DEVELOPER_DIR` prefix:

```bash
DEVELOPER_DIR=/Applications/Xcode.app xcodebuild -project Eyrie.xcodeproj -scheme Eyrie -configuration Debug build
```

Fast iteration happens per package (plain CLT `swift` works, no license/DEVELOPER_DIR needed):

```bash
swift test --package-path Packages/FocusKit                            # one package's tests
swift test --package-path Packages/FocusKit --filter FocusModuleTests  # one suite
for pkg in Packages/*/; do swift test --package-path "$pkg"; done      # all tests
```

Other scripts:

```bash
DEVELOPER_DIR=/Applications/Xcode.app xcodebuild ... -derivedDataPath build build \
  && open build/Build/Products/Debug/Eyrie.app   # run the app
swift Scripts/generate-appicon.swift             # regenerate AppIcon assets
./Scripts/make-dmg.sh                            # Release build + dist/Eyrie-<version>.dmg
```

Release steps are in `RELEASING.md`.

## Architecture

The seam is the `EyrieModule` protocol in `Packages/EyrieCore/Sources/EyrieCore/EyrieModule.swift`. Modules are `@MainActor @Observable` classes exposing `panelContent` / `panelAccessory` / `settingsContent` as `AnyView`; `isActive` drives the menu bar icon (bird → bird.fill). Adding a module = new package + entry in `project.yml` + one line (plus import) in `App/ModuleRegistry.swift`. Enable/disable, the settings tab, and shutdown handling come from the registry and protocol defaults.

Shared services in EyrieCore:
- **PowerAssertionService** — token-based IOKit assertion owner. AwakeKit and FocusKit hold independent tokens; never create IOPM assertions elsewhere.
- **NotificationService** — has a `Bundle.main.bundleIdentifier` guard because `UNUserNotificationCenter` traps in unbundled processes (swift test runners). Keep it.
- **`Date.timerRange(until:)`** — always use this for `Text(timerInterval:)`; a raw `now...end` traps when `end` slips into the past (this was a real shipped crash at phase turnover).
- **ModuleCard / GlassIconButton** — the shared Liquid Glass visual language; module panels render inside `ModuleCard`.

Module-specific invariants:
- **FocusKit**: a naturally finished phase parks in `pendingPhase` until the user taps start (`startPending()`); only explicit `skip()` auto-enters the next phase. This was a deliberate UX decision — do not reintroduce auto-advance. History persists as JSON via `FocusHistoryStore` (injectable `fileURL` for tests).
- **AudioShareKit**: sharing = stacked aggregate device. On stop, restore the previous default output *before* destroying the aggregate. All CoreAudio calls live in `CoreAudioSupport`.
- **DisplayKit**: private `IOAVService` APIs via `@_silgen_name` (the reason this app is direct-distribution only, no sandbox, no App Store). Pure DDC byte protocol lives in `DDCPacket` (unit-tested); `DDCService` is an actor that owns handles and serializes I2C.
- Any module that changes state outside the process must implement `shutdown()`; the `AppDelegate` calls it on quit.

Conventions:
- Swift 6 strict concurrency; modules on `@MainActor`, slow/serial hardware I/O in actors.
- Persistence is `UserDefaults` with `<moduleID>.<key>` naming (e.g. `focus.keepAwake`, `audioshare.selected`).
- UI strings are English, live next to their views (no localization yet).
- macOS 26-only APIs (`glassEffect`, `GlassEffectContainer`, `.buttonStyle(.glass)`) are used without availability fallbacks.

## Tests

Swift Testing (`@Test` / `#expect`), one test target per package. Tests may touch real system APIs but must be read-only or self-cleaning: real power assertions are released within the test, CoreAudio tests never change the default output, and anything file-backed uses an injected temp URL. After test changes, verify no leftovers with `pmset -g assertions | grep -i eyrie`.
